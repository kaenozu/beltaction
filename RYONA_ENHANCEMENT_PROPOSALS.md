# リョナゲー強化提案書

## 1. リョナ要素の現状評価

### 既に実装されている優れたリョナ要素
- **拘束システム**: チェーン拘束、2人掛かりボディブロー
- **ダウン追撃**: ダウン状態での制限付き連続攻撃
- **ポストゲーム攻撃**: ゲームオーバー後のオプション攻撃
- **多層リアクション**: light hurt → heavy hurt → knee-buckle → knockback → down → downhit
- **低HP危険状態**: 動きの低下、長い回復時間

## 2. 新規提案：リョナゲー体験の深化

### A. プレゼンテーション強化

#### 2.1 視覚的インパクト
```
優先度: 高
実装コスト: 中
```

- **ヒットストップの段階化**
  - ライトヒット: 2-3フレーム
  - ヘビーヒット: 5-6フレーム + 白点滅
  - ガードヘッド: 8フレーム + カメラシェイク
  - ダウンヒット: 4フレーム + 画面揺れ
  - チェーン拘束ボディブロー毎: 3フレーム

- **カメラワークの追加**
  - ダウン時: 一時的にズームアウト（全体を tragedi に見せる）
  - ポストゲーム攻撃開始時: ゆっくりズームイン
  - 拘束開始時: 被写界深度風（前景の敵をぼかす）

- **エフェクト強化**
  - ダメージ数値の表示（位置・大きさ・色で差別化）
  - ヒット時のスパーク多重（punch = 白、kick = 橙、heavy = 赤大）
  - 接地パーティクル（ダウン着地時）

#### 2.2 音響デザイン
```
優先度: 中
実装コスト: 低
```

- **打撃音の階層化**
  - light: 軽いパンチ音
  - heavy: 重い衝撃音 + 低音
  - body blow: 腹に響く音
  - down hit: 地面を打つ音
  - grab: 布が裂く/引っ張る音

- **声色の追加**
  - 被弾時: キューイング効果音（短い女性の叫び）
  - ダウン時: 長い断末魔
  - 呼吸音: 低HP時の苦しげな呼吸（ループ）

### B. ゲームプレイ深化

#### 2.3 新規拘束技「串刺しコンボ」
```
優先度: 中
実装コスト: 高
```

**コンセプト**: チェーン拘束中に複数の敵が順番に攻撃する、読みにくくも避けられる数列攻撃

**実装**:
1. 最初の敵がチェーン拘束
2. 2体目の敵が後ろに移動
3. 3体目の敵が真正面から飛び道具/突進
4. プレイヤーは:
   - 早い入力で拘束解除
   - タイミング良くジャンプで飛び道具回避 + 敵の上に着地して反撃
   - 成功すれば周りの敵を巻き込む爆風ダメージ

**バランス**:
- 失敗 = 確実に1発 + 拘持継続
- 成功 = 爽快感 + リスクリワード
- クールタイム: 30秒（同じ敵からは再発動しない）

#### 2.4 「追い詰め」メカニクス
```
優先度: 低
実装コスト: 中
```

**コンセプト**: HPが低いほど敵の行動が積極的になる

**実装**:
- HP 70%以下: 敵の攻撃頻度 +20%
- HP 50%以下: 敵が前より出てくる距離 +30%
- HP 30%以下: チェーン拘束の発生率が通常の2倍
- HP 10%以下: ランダムで1体の敵が「 juggler 」状態（投げ技を連発）

**ビジュアル**:
- HPが下がるたびに背景の色調が暗くなる
- マキの呼吸が荒くなる（パーティクル追加）
- 敵の表情がより残酷になる（既存スプライトの色調変更）

#### 2.5 「見せしめ」システム
```
優先度: 低
実装コスト: 高
```

**コンセプト**: ゲームオーバー直前の数秒だけ特別な演出が入る

**実装**:
1. HPが20%以下になると画面エフェクト追加（ノイズ、色彩低下）
2. 敵の攻撃が1ヒットで確定ダウンするようになる
3. 最後の攻撃が入った瞬間:
   - 速度25%でスローモーション
   - カメラがマキの顔にフォーカス
   - 無音（0.5秒）
   - 衝撃音と同時に通常速度に戻る
4. **オプション**: ポストゲーム攻撃が自動発動する「残酷モード」

### C. 難易度とリプレイ性

#### 2.6 ハードコアモード
```
優先度: 低
実装コスト: 低
```

- 敵の発生数 +50%
- チェーン拘束の逃がす難易度上昇（BOUND_ESCAPE_REQUIRED を 40に）
- 低HP時のステータス低下がより厳しく
- ダウン復帰不能（1回のダウンヒットで死亡）オプション
- スコアランキング実装（クリア時間 + 被弾回数）

### D. メタ的な要素

#### 2.7 敗北演出の多様性
```
優先度: 中
実装コスト: 中
```

**複数のゲームオーバーシナリオ**:

1. **通常敗北**: 標準の死亡演出
2. **拘束敗北**: チェーン拘束中に倒れる + 追加攻撃
3. **連続ダウン敗北**: 3回以上ダウンヒットを受けて死亡
4. **壁際敗北**: 画面端でダウン → 追い打ち
5. **空中敗北**: ジャンプ中に大ダメージ → 画面外に落ちる

**実装方法**:
- `defeatType` フィールドを Player に追加
- それぞれの条件で自動判定
- Game Over 画面で敗北タイプに応じたテキスト/画像表示

## 4. 注意点とバランス

### 守るべき原則
- **プレイヤーエージェンシー**: 全ての強制的な演出には明確な脱出機会を用意
- **オプション設定**: 残酷表現は必ずトグル可能に
- **リスクリワード**: リョナ要素は挑戦する側が自ら選択する構造に
- **フェアネス**: 劣勢でも1つのミスで即死は避ける

### 避けるべき方向
- 見ているだけの強制見せしめ（プレイヤーが操作不能な時間が3秒を超えない）
- ループ可能な無限拘束
- 救いのない難易度（完全無敵時間のない連続攻撃）

## まとめ

このプロジェクトは既にリョナゲームとして極めて高いポテンシャルを持っています。拘束システム、ダウン追撃、ポストゲーム攻撃といったコアループが完成しているため、あとは**段階的なプレゼンテーション強化**と**緊張感の持続**に注力することで、真正なリョナゲーム体験へと昇華できるでしょう。

**最も重要なのは「選択の自由」を保ちつつ、状況を悪化させるリスクと報酬を明確にすること**です。プレイヤーが「あの状況をもう一度見たい」「今度は逃げ切ってみせた」と思える設計が、長期的なゲーム体験を支えます。
